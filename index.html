GIT notes

git init - создает репозиторий в текущей директории

git add - добавляет файлы или директории в "индекс" (облако в котором храняться изменения перед их записью в репозиторий (перед созданием коммита))

git fetch - переносит изменения из виртуального репозитория в локальный (запрос на актуализацию изменений со стороны других разработчиков) Не объединяет изменения с локальным репозиторием

git pull - переносит изменения из виртуального репозитория в локальный, но при этом объединяет их с нашим локальным репозиторием (комбинация двух команд "git fetch" и "git merge")

git push - отправляет коммиты из локального репозитория в удаленный

git branch - создать ветку

git checkout - сменить ветку в которой работаем

git checkout -b - создать и сменить ветку одновременно

Чтобы исключить файлы и директории из репозитория существует специальный файл .gitignore.txt, в котором: # - Комментарий /Name - название папки *.dns - все файлы с указанным расширением

Pull request - это контейнер для ветки в котором мы собираем все изменения перед объединением с главной веткой. Также в Pull request собирает итоговый файл исходя из всех коммитов который были сделаны в ветке
